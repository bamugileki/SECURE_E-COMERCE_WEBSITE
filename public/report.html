1. Introduction

1.1 Background

SmartTrade Africa Ltd is a rapidly growing digital commerce company based in Tanzania that plans to launch a modern e-commerce platform allowing customers to buy products online via both mobile and web applications. The company has identified critical trust and security challenges including fake customer accounts, online payment fraud, unauthorized access attempts, identity theft, customer distrust toward online payments, and data privacy concerns.

1.2 Problem Statement

Traditional e-commerce platforms face significant challenges in building and maintaining customer trust. Security breaches, identity theft, and payment fraud have eroded consumer confidence in online transactions. SmartTrade Africa Ltd requires a comprehensive solution that addresses these challenges through the integration of trusted computing technologies, biometric authentication, secure payment processing, and trust management principles.

1.3 Project Objectives

  • Develop a secure e-commerce web application with comprehensive features
  • Integrate Trusted Computing Platform (TCP) concepts for secure session and device management
  • Implement WebAuthn-based fingerprint biometric authentication
  • Integrate multiple simulated and real payment APIs (Stripe, PayPal, Mock)
  • Apply trust management principles throughout the system
  • Improve customer trust and user confidence through transparent security indicators
  • Apply TAM/UTAUT concepts to improve technology adoption and usability
  • Implement secure software development practices (CSRF, XSS, SQL injection prevention)

1.4 Scope

The application covers user registration and authentication (including biometrics), product catalog management, shopping cart and wishlist functionality, secure checkout with multiple payment gateways, order management, customer reviews and ratings, admin dashboard with analytics, and comprehensive trust and security features.

6. Fingerprint Biometric Authentication (WebAuthn)

6.1 Technology: WebAuthn / FIDO2

The application implements the W3C Web Authentication API (WebAuthn) standard using the lbuchs/webauthn PHP library. WebAuthn is a phishing-resistant authentication standard that allows users to authenticate using biometric sensors (fingerprint, face recognition), platform authenticators (Windows Hello, Apple Touch ID), or external security keys.

6.2 Key Features

  • Biometric Registration: Users enroll their fingerprint via browser's native credential management API
  • Biometric Authentication: One-tap fingerprint login after initial password authentication
  • Credential Management: Users can view, add, and remove enrolled biometric credentials
  • Remote Setup: Token-based biometric enrollment links for device setup
  • Phone Verification: OTP-based phone verification before biometric enrollment
  • Device Support Check: Automatic detection of WebAuthn support on the user's device

6.3 Implementation Details

The biometric system works through a challenge-response protocol:

1. Server generates a cryptographic challenge (random bytes) 2. Client creates a credential using the device's authenticator (fingerprint sensor) 3. Client returns the credential's public key and attestation 4. Server stores the public key associated with the user account 5. On subsequent logins, server challenges the client to sign with the private key 6. Client uses fingerprint to authorize the signing operation 7. Server verifies the signature against the stored public key

7. Trusted Computing Platform (TCP) Integration

7.1 TCP Concepts Applied

TCP ConceptImplementation
Secure Session ManagementDatabase-driven sessions with 120-minute timeout, IP tracking, user agent validation, and session regeneration after login
Device Trust VerificationWebAuthn credential binding ties authentication to specific devices; biometric credentials are device-specific and cannot be copied
Secure Token HandlingCSRF tokens on all forms, signed URLs for email verification, token-based biometric setup links, password reset tokens with expiration
Trusted Login ValidationMulti-factor approach: email/password + optional WebAuthn biometric verification; rate limiting on failed attempts
Secure CommunicationHTTPS enforcement (configurable), all sensitive data transmitted over encrypted channels, HSTS-ready
Secure Credential StoragePasswords hashed with bcrypt (12 rounds), WebAuthn public keys stored securely, no plain-text secrets
Integrity MeasurementActivity logging tracks all user actions for audit; payment status is verified end-to-end

7.2 Session Security

The session driver is set to database for persistence and auditability. Session lifetime is 120 minutes. The session ID is regenerated after login to prevent session fixation attacks. Each session record in the database includes the user ID, IP address, user agent, and last activity timestamp for monitoring.

7.3 Token-Based Security

The application uses multiple token mechanisms:

  • CSRF Tokens: Every form includes a CSRF token validated by Laravel's middleware
  • Email Verification Tokens: Signed URLs for verifying email addresses
  • Password Reset Tokens: Hashed tokens stored in the password_resets table
  • Biometric Setup Tokens: 64-character random tokens for remote WebAuthn enrollment
  • API Tokens: Session-based authentication for AJAX requests

10. TAM / UTAUT Integration

10.1 Technology Acceptance Model (TAM)

The application is designed following the Technology Acceptance Model (TAM), focusing on two key constructs:

a) Perceived Usefulness (PU)

The application helps customers complete shopping activities efficiently:

  • Fast Checkout: Streamlined multi-gateway checkout process with minimal steps
  • Quick Payment Confirmation: Real-time payment status updates and instant order confirmation notifications
  • Efficient Product Search: Category filtering and keyword search with paginated results
  • Helpful Notifications: Real-time database notifications for order confirmations and updates
  • Product Discovery: Category browsing, search, and wishlist management

b) Perceived Ease of Use (PEOU)

The system is simple and easy to use:

  • Clear Navigation: Bootstrap 5 navbar with desktop/mobile responsive layouts, breadcrumb-style back button, and offcanvas mobile menu
  • Simple Registration Process: Minimal fields (name, email, password, phone) with clear validation messages
  • Easy Fingerprint Login: One-tap biometric authentication after initial setup
  • User-Friendly Interface: Tailwind CSS with consistent design, hover effects, loading animations, and toast notifications
  • Responsive Design: Mobile-first approach with offcanvas navigation and adaptive grid layouts

10.2 Unified Theory of Acceptance and Use of Technology (UTAUT)

UTAUT constructs are also addressed:

a) Performance Expectancy

  • Faster Payment Processing: Multi-gateway support with instant mock payment confirmation or real Stripe/PayPal processing
  • Reliable Transactions: Database transaction safety, payment verification, stock management integrity
  • Secure Customer Experience: Visible security badges, biometric authentication, and trust indicators

b) Effort Expectancy

  • Easy Checkout Process: Gateway selection with clear descriptions, minimal clicks to complete purchase
  • Simple App Navigation: Consistent layout, clear menu structure, search functionality
  • Easy Biometric Authentication: One-click fingerprint login after initial enrollment

c) Social Influence

  • Customer Reviews: Star ratings and written reviews on all product pages
  • Testimonials: Dashboard showcases high-rated reviews from real customers
  • Social Media Integration: Share products on WhatsApp, Facebook, and Twitter
  • WhatsApp Inquiry: Direct product inquiry via WhatsApp business integration

d) Facilitating Conditions

  • Mobile Responsiveness: Full responsive design using Bootstrap 5 + Tailwind CSS
  • Cross-Platform Compatibility: Works on all modern browsers (Chrome, Firefox, Safari, Edge) and devices
  • Stable System Performance: Optimized database queries, eager loading, Vite asset bundling
  • Offline-Ready: Toast notifications and form validation work without page reloads

14. Conclusion & Recommendations

14.1 Conclusion

The SmartTrade Africa Ltd Secure E-Commerce Platform has been successfully developed as a comprehensive solution addressing the trust and security challenges faced by modern e-commerce businesses. The application integrates:

  • Trusted Computing Platform (TCP) concepts including secure session management, device trust verification, secure token handling, and credential protection

14.2 Recommendations

  1. Production Deployment: Migrate from SQLite to MySQL/PostgreSQL for production scalability and performance
  2. Real SMS OTP: Integrate an SMS gateway (Twilio, Africa's Talking) for actual phone verification OTP delivery
  3. Multi-Factor Authentication: Add TOTP (Google Authenticator) or SMS-based 2FA as an additional security layer
  4. Flutterwave Gateway: Complete the Flutterwave integration to support African mobile money payments (M-Pesa, Airtel Money)
  5. Machine Learning Fraud Detection: Implement ML-based fraud detection for unusual transaction patterns
  6. Mobile Applications: Develop native Android/iOS applications with native biometric APIs for enhanced fingerprint authentication
  7. Blockchain Integration: Explore blockchain-based trust scoring for tamper-proof transaction history
  8. Performance Optimization: Implement Redis caching, database indexing, and CDN for static assets
  9. Penetration Testing: Regular security audits and penetration testing by third-party security firms
  10. User Trust Surveys: Conduct regular TAM/UTAUT surveys to measure technology acceptance and identify improvement areas
